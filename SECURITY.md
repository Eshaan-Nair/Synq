# Security Policy

## Supported Versions

Currently, only the `main` branch is supported with security updates. 

| Version | Supported          |
| ------- | ------------------ |
| main    | :white_check_mark: |

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability in SYNQ, please do not disclose it publicly.

Instead, please report it via [GitHub Security Advisories](https://github.com/Eshaan-Nair/Synq/security/advisories/new) or reach out to the maintainer directly — contact details are available on the [GitHub profile](https://github.com/Eshaan-Nair).

When reporting a vulnerability, please include:
* A detailed description of the vulnerability.
* Steps to reproduce the issue.
* The environment in which the vulnerability was found (OS, browser, Node version).
* Potential impact of the vulnerability.

We will acknowledge receipt of your vulnerability report within **48 hours**. We target resolution within **10 days** for critical issues and **15 days** for moderate issues. You will receive regular updates on progress throughout. If you have not received a reply within 48 hours, please follow up.
